# C4 功能描述文档

> **版本**：v0.2.5 | **最后更新**：2026-07-16 | **来源文档**：`c4_requirement.md` v0.3.0

---

## 1. 用户交互

### C4_FUN_00001：Agent 接受并理解用户自然语言指令

**功能描述**：

Agent 接受用户以自然语言提交的数据接入需求，理解用户的数据接入意图，
从指令中提取关键信息（设备类型、通信参数、数据点范围等），并据此执行后续的数据接入工作。

**实现的需求**：C4_RS_00040, C4_RS_00005, C4_RS_00044, C4_RS_00001

---

### C4_FUN_00002：Agent 解析结构化文档并提取关键信息

**功能描述**：

Agent 接收并解析结构化文档（如 Excel 点表、CSV 配置表），
从中自动提取设备地址、寄存器映射、数据类型、通信参数等关键信息。

**实现的需求**：C4_RS_00041, C4_RS_00044

---

### C4_FUN_00003：Agent 解析非结构化文档并提取关键信息

**功能描述**：

Agent 接收并解析非结构化文档（如 PDF 设备手册、Word 规范文档、现场照片和截图），
从中自动提取设备地址、寄存器映射、数据类型、通信参数等关键信息。

**实现的需求**：C4_RS_00042, C4_RS_00044

---

### C4_FUN_00004：Agent 从输入中生成可执行的数据接入方案

**功能描述**：

Agent 从已解析的用户输入（自然语言、结构化文档、非结构化文档或混合组合）中提取数据接入意图，
生成一份包含通信协议、目标设备、数据点清单和转发目标的可执行方案，供用户确认。

**实现的需求**：C4_RS_00050, C4_RS_00043, C4_RS_00001

---

### C4_FUN_00005：Agent 使用非技术语言进行交互

**功能描述**：

Agent 向用户呈现的提示、错误信息和诊断报告使用非技术语言，
避免协议级术语和内部错误码。在用户提供信息不完整时，Agent 主动引导用户补充缺失的关键信息。

**实现的需求**：C4_RS_00260, C4_RS_00261

---

### C4_FUN_00041：C4 通过 Web 界面与用户交互

**功能描述**：

C4 通过 Web 界面与用户交互，用户可在浏览器中提交数据接入需求、
上传配置文档、查看 MCP 服务运行状态和诊断报告。

**实现的需求**：C4_RS_00045

---

## 2. 任务编排

### C4_FUN_00006：Agent 管理 MCP 服务的生命周期

**功能描述**：

Agent 通过 Model Context Protocol 与 MCP 服务通信，执行启动、配置和停止操作，
启动后自动完成协议级参数配置。MCP 服务按需启动，由 Agent 管理其完整生命周期。

**实现的需求**：C4_RS_00022, C4_RS_00025, C4_RS_00052, C4_RS_00053, C4_RS_00001

---

### C4_FUN_00007：Agent 自主执行常规操作

**功能描述**：

Agent 对常规操作——包括启动已知协议的 MCP 服务、执行标准状态监控、
恢复已知类型的故障——自主执行，无需每次人工介入。

**实现的需求**：C4_RS_00033, C4_RS_00001

---

### C4_FUN_00044：Agent 将接入方案分解为可执行步骤

**功能描述**：

Agent 将用户确认的接入方案分解为可执行的步骤序列——包括选择对应的通信协议、
确定具体的配置参数、识别需要启动的 MCP 服务类型及其启动顺序。

**实现的需求**：C4_RS_00051

---

## 3. 数据接入

### 3.1 数据接入

### C4_FUN_00008：MCP 服务执行确定性的数据采集

**功能描述**：

MCP 服务接收 Agent 的配置指令，执行确定性的数据采集工作。
同一配置和输入下，MCP 服务始终产生相同的采集结果。

**实现的需求**：C4_RS_00080, C4_RS_00081, C4_RS_00034

---

### C4_FUN_00009：MCP 服务执行数据转换

**功能描述**：

MCP 服务在数据流转过程中执行数据转换工作——包括协议格式转换、
数值单位换算和数据结构重组。

**实现的需求**：C4_RS_00085

---

### C4_FUN_00010：MCP 服务执行确定性的数据转发

**功能描述**：

MCP 服务按 Agent 的配置将已采集或转换后的数据转发到指定的目标系统。
同一配置和输入下，MCP 服务始终产生相同的转发结果。

**实现的需求**：C4_RS_00082, C4_RS_00034

---

### C4_FUN_00011：MCP 服务向 Agent 报告运行状态

**功能描述**：

MCP 服务在执行过程中持续向 Agent 报告运行状态、数据统计和异常信息。

**实现的需求**：C4_RS_00083

---

### C4_FUN_00012：C4 支持 Modbus 协议数据采集

**功能描述**：

C4 通过 Modbus TCP/RTU 协议采集工业设备的寄存器数据。

**实现的需求**：C4_RS_00090

---

### C4_FUN_00013：C4 支持 IEC 60870-5-104 协议数据采集

**功能描述**：

C4 通过 IEC 60870-5-104 协议采集远动设备的数据。

**实现的需求**：C4_RS_00091

---

### C4_FUN_00014：C4 支持 IEC 60870-5-101 协议数据采集

**功能描述**：

C4 通过 IEC 60870-5-101 协议采集串口远动设备的数据。

**实现的需求**：C4_RS_00092

---

### C4_FUN_00015：C4 支持数据转发给第三方系统

**功能描述**：

C4 将采集到的数据按指定格式转发给第三方系统。

**实现的需求**：C4_RS_00093

---

### C4_FUN_00016：C4 支持数据写入数据库

**功能描述**：

C4 将采集到的数据写入指定的数据库。

**实现的需求**：C4_RS_00094

---

### C4_FUN_00017：新协议通过新增 MCP 服务扩展

**功能描述**：

新的工业通信协议的数据接入支持通过新增遵循标准 MCP 接口的 MCP 服务实现，
无需改动 Agent 或已部署的其他 MCP 服务。

**实现的需求**：C4_RS_00024, C4_RS_00023

---

### C4_FUN_00042：C4 接收 ASFP2 数据

**功能描述**：

ASFP2 接收 MCP 服务作为服务端监听连接，接收来自其他 C4 实例或兼容系统的
ASFP2 数据，解析后写入共享内存的对应 point_id 记录。

**实现的需求**：C4_RS_00095

---

### C4_FUN_00047：C4 发送 ASFP2 数据

**功能描述**：

ASFP2 发送 MCP 服务从共享内存读取已订阅 point 的数据，按 ASFP2 协议规范编码为数据包，
发送到中心侧或其他 C4 实例。打包时自动检测属性开关条件（SAME_DATA_TYPE / KEY_SEQUENCE /
SAME_TIMESTAMP）以优化带宽。

**实现的需求**：C4_RS_00095

---

### 3.2 共享内存管理

### C4_FUN_00053：C4 创建共享内存空间 ✅

**功能描述**：

C4 启动时根据配置文件创建共享内存空间。如果配置文件不存在或为 null，创建并初始化一个容纳 10 万点的共享内存空间。如果配置文件存在，根据配置文件中配置的点数创建并初始化 2 倍数量的共享内存空间。

**实现的需求**：C4_RS_00096

---

### C4_FUN_00054：C4 扩展共享内存空间 ✅

**功能描述**：

C4 收到扩容命令后，根据配置文件计算需要的点数。如果点数没有超过当前共享内存的容量，就在剩余的空间中分配新点。如果点数超过当前共享内存的容量，将内存容量扩大为新点数的 2 倍，并保证已有点地址的数据不被破坏。

**实现的需求**：C4_RS_00096

---

### C4_FUN_00055：C4 回收共享内存空间 ✅

**功能描述**：

C4 收到调整命令后，根据配置文件计算需要的点数。如果配置中的点数少于当前共享内存中已激活的点数，则将被删除的点占用的 block 的 state 置 0 完成回收，并保证未被删除的点的空间不被破坏。回收后的空闲 block 可供新增点复用。

**实现的需求**：C4_RS_00096

---

### C4_FUN_00056：C4 再分配共享内存空间

**功能描述**：

C4 收到调整命令后，根据配置文件计算需要的点数。在完成回收（如有）且无需扩容的情况下，或扩容完成之后，将当前空余的地址空间（包括回收后产生的空闲 block）分配给新增加的点，并保证现有点的地址空间不被破坏。

**实现的需求**：C4_RS_00096

---

### C4_FUN_00049：C4 管理共享内存生命周期

**功能描述**：

共享内存管理 MCP 服务是每个 C4 实例默认启动的首个 MCP 服务，负责 POSIX 共享内存的创建、扩容、块分配回收和状态查询。

**实现的需求**：C4_RS_00096

---

## 4. 监控自愈

### C4_FUN_00018：Agent 持续监控 MCP 服务运行状态

**功能描述**：

Agent 持续监控所有已启动 MCP 服务的运行状态，并监控采集成功率、
数据时效性和数据完整性等数据采集指标。当指标偏离正常范围时触发异常处理流程。

**实现的需求**：C4_RS_00060, C4_RS_00061

---

### C4_FUN_00019：Agent 智能诊断异常并生成诊断报告

**功能描述**：

Agent 在发现异常时分析根因，生成包含异常现象描述、可能原因列表和
建议操作步骤的诊断报告。当异常为涉及数据接入配置的变更时，报告升级为人工审核。

**实现的需求**：C4_RS_00062, C4_RS_00063, C4_RS_00065, C4_RS_00001

---

### C4_FUN_00020：Agent 自动修复已知故障

**功能描述**：

Agent 对可自动处理的已知故障（如通信连接中断）执行自动修复操作，
无需人工介入。修复完成后 Agent 自动恢复对 MCP 服务的监控。

**实现的需求**：C4_RS_00064

---

### C4_FUN_00021：MCP 服务异常退出后自动重启

**功能描述**：

MCP 服务在异常退出后自动重启，恢复数据采集和转发工作，
无需人工干预。

**实现的需求**：C4_RS_00240

---

### C4_FUN_00022：Agent 故障不影响 MCP 数据管道

**功能描述**：

Agent 故障（进程崩溃、LLM 不可达、重启）不影响已启动并正在运行的 MCP 数据管道。
MCP 服务在 Agent 不可用期间以守护进程模式继续自主执行数据采集和转发，
不中断数据流。AI 不进入实时数据路径，确保数据接入的可靠性和确定性。

**实现的需求**：C4_RS_00030, C4_RS_00031, C4_RS_00002, C4_RS_00003

---

### C4_FUN_00023：Agent 恢复后自动接续监控

**功能描述**：

Agent 故障恢复后自动接续对所有已运行 MCP 服务的监控状态，无需人工干预。

**实现的需求**：C4_RS_00032

---

### C4_FUN_00024：C4 以守护进程持续运行

**功能描述**：

C4 以守护进程模式持续运行，在资源可用的情况下不因自身原因退出或停止响应。
MCP 服务在 Agent 不可用期间继续执行数据采集和转发。

**实现的需求**：C4_RS_00014, C4_RS_00084

---

### C4_FUN_00043：C4 按故障等级发送告警通知

**功能描述**：

C4 对达到设定故障等级的异常事件，通过邮件或其他指定渠道向相关运维人员
发送告警通知。故障等级阈值和通知渠道支持配置。

**实现的需求**：C4_RS_00068

---

### C4_FUN_00045：Agent 在数据源接入完成后通知用户

**功能描述**：

Agent 在新数据源接入流程全部完成、数据采集和转发均已确认正常后，
主动向用户发送接入完成通知。

**实现的需求**：C4_RS_00067

---

### C4_FUN_00046：C4 在部分 MCP 故障时保持其余服务运行

**功能描述**：

当部分 MCP 服务发生故障时，Agent 和其余正常运行的 MCP 服务不受影响，
各自继续执行已配置的数据采集和转发工作（优雅降级）。

**实现的需求**：C4_RS_00242

---

## 5. 跨子网协同

### C4_FUN_00025：C4 Agent 跨子网构成统一网络

**功能描述**：

部署在不同安全分区（I 区、II 区、III 区、中心侧）的 C4 Agent 通过直接对等通信
构成统一的 C4 网络，不依赖统一管理节点。Agent 之间仅交换任务协调信息和数据流元数据，
不传输原始采集数据。

**实现的需求**：C4_RS_00100, C4_RS_00101, C4_RS_00102

---

### C4_FUN_00026：原始数据由 MCP 服务按配置流向跨区传输

**功能描述**：

原始采集数据的跨安全分区传输由各子网的 MCP 服务按 Agent 配置的数据流向执行，
不经过 Agent 之间的通信通道。

**实现的需求**：C4_RS_00103

---

### C4_FUN_00027：数据端到端路径可追溯

**功能描述**：

C4 网络保证每条数据从生产端（场站设备）到消费端（中心侧数据库或第三方系统）
的路径可追溯——可确定数据的产生设备、经过的子网节点和到达中心侧的时间。

**实现的需求**：C4_RS_00104

---

## 6. 安全防护

### C4_FUN_00028：C4 提供用户身份认证

**功能描述**：

C4 验证访问 Agent 的用户身份，并支持不同 C4 实例的 Agent 之间通过互认身份建立信任。

**实现的需求**：C4_RS_00110, C4_RS_00111

---

### C4_FUN_00029：C4 支持基于角色的操作授权

**功能描述**：

C4 基于角色控制用户的操作权限，关键配置变更须经授权审核。
MCP 服务遵循最小权限原则——仅拥有执行其数据接入任务所需的最小权限。

**实现的需求**：C4_RS_00112, C4_RS_00120

---

### C4_FUN_00030：C4 对跨安全分区通信链路进行加密

**功能描述**：

C4 对跨安全分区（I/II/III 区 / 中心侧）的所有通信链路进行加密传输。

**实现的需求**：C4_RS_00113

---

### C4_FUN_00031：C4 记录操作审计日志

**功能描述**：

C4 记录关键操作——包括配置修改、MCP 服务启停、数据转发配置变更——
以支持事后审查和追溯。

**实现的需求**：C4_RS_00114

---

### C4_FUN_00032：C4 防范 LLM 安全攻击

**功能描述**：

Agent 对用户输入和上传文档进行安全校验，防范提示注入攻击；
在生成 MCP 配置指令前对配置内容进行安全校验，防止恶意配置被执行；
向用户输出的内容进行安全过滤，防止泄露系统内部信息或敏感数据。

**实现的需求**：C4_RS_00230, C4_RS_00231, C4_RS_00232

---

### C4_FUN_00033：配置变更须经人工审核

**功能描述**：

任何涉及数据接入配置的变更，须经人工审核确认后方可生效。
Agent 不得在未经审核的情况下修改数据接入配置。

**实现的需求**：C4_RS_00066, C4_RS_00121

---

### C4_FUN_00034：C4 遵守安全分区与合规要求

**功能描述**：

C4 在已有安全边界内运行，跨子网通信符合场站网络安全分区管理规定，
不替代场站已有的安全基础设施（防火墙、网闸、入侵检测等）。
C4 的部署支持其运行行业适用的工业网络安全标准合规要求。

**实现的需求**：C4_RS_00122, C4_RS_00123, C4_RS_00004, C4_RS_00270

---

## 7. 部署运维

### C4_FUN_00035：C4 部署于多层安全分区环境

**功能描述**：

C4 部署在工业现场的数据服务器上，运行于有网络隔离的多层安全分区环境
（I/II/III 区及中心侧），与工业设备通信协议交互，同时对接 IT 系统。

**实现的需求**：C4_RS_00010, C4_RS_00011, C4_RS_00012, C4_RS_00013

---

### C4_FUN_00036：C4 生成结构化运行日志

**功能描述**：

Agent 和 MCP 服务生成结构化运行日志，支持可配置的日志级别和日志轮转。

**实现的需求**：C4_RS_00250

---

### C4_FUN_00037：C4 提供健康检查端点

**功能描述**：

C4 提供健康检查端点，供外部运维系统对 C4 各组件的运行状态进行监控。

**实现的需求**：C4_RS_00253

---

### C4_FUN_00038：C4 支持滚动升级

**功能描述**：

MCP 服务可在数据管道持续运行期间进行升级；
Agent 可在不中断已运行 MCP 服务的情况下进行升级。

**实现的需求**：C4_RS_00251, C4_RS_00252

---

## 8. 数据治理

### C4_FUN_00039：C4 数据所有权归属场站方

**功能描述**：

用户上传的配置文档和经由 C4 流转的数据，所有权归属于场站方。

**实现的需求**：C4_RS_00130

---

### C4_FUN_00040：C4 不持久化采集数据

**功能描述**：

C4 不持久化存储采集的数据内容——所有数据经 MCP 管道流式转发，
不在 C4 内部的任何环节落地存储。

**实现的需求**：C4_RS_00131

---

## 9. 许可管理

### C4_FUN_00050：Agent 验证许可密钥

**功能描述**：

Agent 启动时读取本地许可文件或环境变量中的许可密钥，使用内置公钥验证其数字签名、
有效期和绑定的硬件指纹。验证通过后按许可等级启用对应功能；验证失败时拒绝启动或降级运行。

**实现的需求**：C4_RS_00300, C4_RS_00305

---

### C4_FUN_00051：Agent 许可激活

**功能描述**：

Agent 支持两种激活方式：在线激活时连接授权服务器，提交硬件指纹获取绑定本机的许可密钥；
离线激活时用户将从授权服务器获取的许可文件导入 Agent，由 Agent 本地验证后完成激活。

**实现的需求**：C4_RS_00301, C4_RS_00302

---

### C4_FUN_00052：Agent 许可到期管理

**功能描述**：

Agent 在许可到期前 7 天开始向运维人员发送告警通知；许可到期后进入 7 天宽限期，
期间保持全部正常功能；超宽限期后按许可等级限制可用功能——包括 MCP 服务实例数上限、
禁用跨子网协同和高级监控等功能——或完全停止服务。

**实现的需求**：C4_RS_00303, C4_RS_00304
